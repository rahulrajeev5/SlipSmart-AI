# SlipSmart AI – User Stories

## Purpose

User stories describe the functionality of SlipSmart AI from the user's perspective. They help ensure every feature delivers value to the end user and guide development throughout the project.

---

# Epic 1 – View Matches

### US-001

**As a sports bettor, I want to view upcoming football matches so that I know which matches are available for prediction analysis.**

---

### US-002

**As a sports bettor, I want to view match details including league, teams, kickoff time, and status so that I can make informed decisions before generating betting slips.**

---

# Epic 2 – Predictions

### US-003

**As a user, I want to view AI-generated predictions so that I can understand the most likely outcomes for upcoming matches.**

---

### US-004

**As a user, I want to see prediction probabilities and confidence levels so that I can better evaluate the associated risk.**

---

### US-005

**As a user, I want to compare different prediction markets so that I can choose the betting strategy that best fits my preferences.**

---

# Epic 3 – Slip Generator

### US-006

**As a user, I want to enter my betting budget so that the generated slips distribute my stake correctly.**

---

### US-007

**As a user, I want to choose a betting strategy (Safe, Balanced, or Aggressive) so that the generated slips match my risk tolerance.**

---

### US-008

**As a user, I want the system to generate multiple diversified betting slips so that a single incorrect prediction does not affect every slip.**

---

### US-009

**As a user, I want to see the total odds, estimated probability, and potential return for each generated slip so that I can compare different betting options.**

---

# Epic 4 – AI Assistant

### US-010

**As a user, I want the AI to explain why specific predictions were selected so that I understand the reasoning behind each recommendation.**

---

### US-011

**As a user, I want to ask questions about predictions so that I can better understand teams, players, and betting strategies.**

---

# Epic 5 – User Management

### US-012

**As a user, I want to create an account so that I can save my betting history and preferences.**

---

### US-013

**As a user, I want to save generated betting slips so that I can review them later before placing bets.**

---

# Epic 6 – Future Features

### US-014

**As a user, I want to receive notifications before matches begin so that I never miss an opportunity to review my predictions.**

---

### US-015

**As a user, I want to compare my prediction performance over time so that I can improve my betting strategy.**

---

# Acceptance Criteria

A user story is considered complete when:

* The functionality works as expected.
* The user can successfully complete the intended task.
* Validation and error handling are implemented.
* The feature is tested.
* Documentation is updated.
